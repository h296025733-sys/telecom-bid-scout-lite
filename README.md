# telecom-bid-scout-lite

**通信行业招采线索筛选与飞书推送小工具**

`telecom-bid-scout-lite` 用于处理已经整理好的通信行业候选招采公告。它读取 CSV 候选数据和 YAML 规则，输出可复核的筛选结果、Markdown 报告，并按需推送到飞书。

这个项目关注一个可运行的轻量闭环：先把候选线索筛清楚，再交给人工确认是否继续跟进。

## 解决的问题

通信行业招采线索里常混有过期公告、转载页面和与通信业务无关的采购内容。只看关键词容易带来噪音，逐条人工筛选又比较费时。

本项目当前会结合以下信息做规则判断：

- 通信方向关键词与排除词；
- 发布时间、截止时间和来源可信度；
- 候选链接与官方公告入口；
- 已处理状态，避免重复推送。

每条结果都会保留评分、命中关键词和判断原因，便于复核筛选逻辑。

## 核心流程

```text
候选公告 CSV
  -> 加载 YAML 规则
  -> 规则评分与原因生成
  -> 值得跟进 / 需人工复核 / 不建议跟进
  -> Markdown 报告
  -> 可选飞书推送
  -> state.json 去重记录
```

## 快速运行

要求 Python 3.10+。

```bash
python -m pip install -r requirements.txt
python main.py run --no-feishu
```

运行后可查看：

- `outputs/report.md`
- `outputs/state.json`

需要启用飞书推送、重置状态或查看更完整的安装说明，见 [docs/usage.md](docs/usage.md)。

## 示例结果

报告会把候选线索分级，并解释判断依据：

```markdown
### 1. 某市政务热线呼叫中心坐席扩容及外呼系统采购公告

- 地区：广东深圳
- 来源：公共资源交易中心
- 判断结果：**值得跟进**
- 评分：100
- 命中关键词：呼叫中心、外呼系统
- 判断原因：命中通信方向关键词；来源可信；仍在有效期内...
```

当前样例报告见 [outputs/report.md](outputs/report.md)。

## 项目边界

本项目是通信行业招采线索的规则筛选器，不是完整招采平台，也不是事实核验系统。

它当前适合做：

- 对候选公告做轻量分级和人工复核前置；
- 用可解释规则辅助通信招采线索筛选；
- 生成报告、记录状态，并可选发送飞书提醒。

它当前不替代：

- 搜索全网招采信息和网页正文抓取；
- 对资质、预算、投标主体限制做最终判断；
- 正式报名、投标和商务跟进动作。
